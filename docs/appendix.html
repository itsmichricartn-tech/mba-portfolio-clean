B) Prompts I Provided to the LLM (Selected, Verbatim)

B1. Global Structure Prompt

You are a UX-savvy writing assistant helping me build a clean, professional multi-page MBA portfolio with Quarto. Deliverables:

-A site map with 6–7 pages: Home, Resume, Projects, Skills & Certifications, Contact, Reflection, Appendix.

-A Quarto navbar outline (file names like index.qmd, resume.qmd, etc.).

-One 10–15 word description per page.Style: concise, practical, business-ready; avoid hype. Output in Markdown only.

B2. Home/About Tone Prompt

You are a friendly, UX-savvy writing assistant. Rewrite a short Home/About intro (3–5 sentences). Goals: approachable, competent, people-centric HR + current culinary practice in Vancouver. Must include: name, MBA Candidate at University Canada West (Vancouver, BC); strengths in Compensation & Benefits, employee services, onboarding; values of clarity, service, and reliable processes; CTAs: “View Resume”, “View Projects”, “Download PDF”. Output: ≤2 short paragraphs, no emojis, no hype.

B3. Resume Tightening Prompt

Tighten resume bullets for these roles (keep employer/location/dates as given):

  • Compensation & Benefits Analyst — Shell Licensee (Santo Domingo, DR) (Dec 2021–Sep 2024)

  • General Analyst — Shell Licensee (Dec 2021–Aug 2024)

  • Cook — Guilt and Co (Vancouver, Oct 2025–Present)

  • Line Cook — Six Acres (Mar–May 2025)

  • Plater — Fed (Mar–Sep 2025)

  • Stock Service — ZARA (Nov 2019–Nov 2021)

Rules: 2–3 bullets per role; action-first verbs; clear outcome/impact where possible; ≤140 characters per bullet; no fluff.

Output only Markdown lists under each role title.

B4. Projects Formatting Prompt

Turn each project into a compact card with three fields:

• Overview (1–2 sentences)

• Role (my contribution)

• Outcome (practical result)

Return plain HTML with three div class=“card” blocks that drop into a Quarto page.

B5. Skills/Badges Prompt

-Return a single Markdown block with inline “badges” for core HR and professional skills.

-Format each badge as: <span class="badge">Skill Name</span> separated by spaces/newlines.

-Include: Compensation & Benefits, Employee Services, Onboarding, Internal Communications, Excel (dashboards), Time Management, Problem-Solving, Teamwork, Customer Service, Spanish & English.

-Do not add explanations—badges only.

B6. Reflection Prompt

-Write a 300–500 word reflection about building a multi-page MBA portfolio with Quarto and LLMs.

-Cover: (1) what improved when adding pages, (2) how you verified authenticity/accuracy, (3) limits/risks of LLMs and how you mitigated them (citations, manual review, dated notes, style control), (4) what you’d iterate next (accessibility, performance, content depth).

-Tone: honest, specific, first person. Output in Markdown (no title).

E) Change Log & Rationale

E1. Major edits across drafts

  • Home: Replaced long intro with a concise value statement; added clear CTAs (View Resume, Projects, Download PDF).

  • Resume: Tightened bullets to action-first outcomes; limited to 2–3 bullets/role; removed filler.

  • Projects: Standardized 3-card layout (Overview · Role · Outcome) for quick scanning.

  • Skills & Certifications: Switched to compact “badges” for readability on mobile.

  • Contact: Simplified to essentials (location, email, LinkedIn, resume link).

  • Theme/UX: Added soft gradient background, improved spacing, and consistent heading scale.

E2. Why these changes? (Rationales)

  • Clarity & scannability: Recruiters skim; compact sections and consistent patterns reduce cognitive load.

  • Outcome focus: Bullets emphasize impact, not tasks—clearer value for HR roles.

  • Accessibility: Larger line-height, badge contrast, and predictable navigation improve readability.

  • Authenticity: All facts (work history, skills) were verified and kept in my own voice.

E3. What I did NOT include (and why)

  • Leadership/Community/Publication pages: Deferred to future iterations to keep scope manageable and maintain quality.

  • Long-form narrative: Avoided long paragraphs in Resume/Projects to favor scannable content.

E4. Risks & mitigations

  • Risk: Visual polish might distract from content.
    Mitigation: Kept color accents subtle, prioritized typography and spacing.

  • Risk: Overfitting to one audience.
    Mitigation: Kept language professional-general; industry specifics live inside project cards.

E5. Evidence of authorship

  • All data points (dates, roles, responsibilities) map to my CV and personal records.

  • Prompts and selected outputs are archived in this Appendix (Sections B–C).

  • Local build logs and commit history show progressive edits (see Git history timeline).
